# RedisViewer 隐私政策

更新日期：2026-05-20

RedisViewer 尊重你的隐私。本政策说明 RedisViewer 如何处理匿名基础统计数据，以及哪些数据不会被收集。

## 1. 匿名统计默认关闭

RedisViewer 的匿名统计默认关闭。只有在你点击“接受”或在偏好设置中打开“允许收集匿名数据”后，才会启用匿名统计。

你可以随时在 RedisViewer 的“偏好设置 -> 隐私策略”中关闭该选项。关闭后，RedisViewer 会停止发送后续匿名统计事件。

## 2. 我们收集哪些匿名基础数据

在你同意后，RedisViewer 可能会通过 Umami 收集以下匿名基础统计数据，用于了解兼容性、排查版本问题和改进产品体验：

- 应用版本号
- 操作系统类型
- CPU 架构
- 应用语言
- 启动事件
- Redis 连接数量（仅数量，不包含连接地址或连接名称）
- WebDAV 同步是否启用（仅 `true` 或 `false`，不包含服务地址、账号、目录或同步组名）

RedisViewer 不会主动向统计事件中加入你的姓名、邮箱、账号标识或其他直接身份信息。

## 3. 我们不会收集哪些数据

RedisViewer 不会通过匿名统计收集以下内容：

- Redis 连接地址、连接名称、用户名、密码
- 数据库名称、Key、Value、命令内容
- WebDAV 服务地址、账号、密码、远程目录、同步组名
- 本地文件内容、目录内容、剪贴板内容
- 任何 Redis 数据库中的业务数据

## 4. 使用目的

匿名基础统计数据仅用于：

- 了解 RedisViewer 在不同系统和版本中的兼容性
- 判断功能使用状态，帮助安排产品改进优先级
- 发现启动、版本和平台相关的问题

我们不会出售匿名统计数据，也不会将其用于广告定向。

## 5. 第三方统计服务

RedisViewer 使用 Umami 作为匿名统计服务。统计脚本来自：

`https://cloud.umami.is/script.js`

RedisViewer 已关闭 Umami 的自动页面追踪，仅在你同意后发送必要的匿名统计事件。

和所有网络请求一样，Umami 服务在接收请求时可能会处理必要的技术请求信息，例如 IP 地址、User-Agent、请求时间等。RedisViewer 不会把这些信息作为自定义统计数据主动上报。

## 6. 数据控制与撤回

你可以通过以下方式控制匿名统计：

- 首次提示中点击“不接受”：不会启用匿名统计
- 首次提示中点击“接受”：启用匿名统计
- 在“偏好设置 -> 隐私策略”中关闭开关：停止后续匿名统计事件

撤回同意不会影响你继续使用 RedisViewer。
